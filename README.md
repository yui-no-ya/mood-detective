# 🌙 Mood Detective

気分の波の要因を、気圧・周期・飲酒・睡眠・服薬（デパス）などと突き合わせて
データで特定するための個人ツール。「夜の帳」世界観のゴシックなUI。

最終目標：データが貯まったら、Claude がフラットに分析して「この要因が効いてそう」と
指摘し、それを元に予報ロジックをパーソナライズする。

## 構成

- **アプリ本体**：`index.html` 単一ファイル（Chart.js + バニラ JS）。React / ビルド不要。
- **データ母艦**：Google スプレッドシート（フォーム回答シート）。全記録がここに集約。
  複数端末から入力しても同期される。
- **読み書きの口**：GAS Web アプリ（`/exec`）。`doGet`=読み込み、`doPost`=書き込み。
  アプリの保存ボタンが `doPost` でシートに書き、開くときに `doGet` で読む。
  `/exec` URL はアプリ内「ヘルスケア連携」欄に一度貼ると localStorage に保存される。
- **気圧**：アプリが Open-Meteo から自動取得（小田原 lat 35.2646 / lon 139.1533、海面気圧）。
  気圧はシートには入らず、アプリ側で別途取得して重ねる。

2. `git add -A && git commit -m "変更内容" && git push`
3. 数十秒〜1分で GitHub Pages の公開 URL に自動反映される。

手動アップロードは不要。これが「差し替え地獄」を解消する仕組み。


## アプリの主要仕様

- 入力項目：気分(1–5)、睡眠(h)、デパス(0/1)、飲酒(0–3)、体調(0–3)、喉(0–3)、
  症状(自由文)、生理初日(🩸トグル)、メモ、運動。
- 周期は「生理初日を一度マークすれば、以降の周期◯日目を自動計算」。毎日入力しない。
- 手貼り取り込み（1行1日）例：`2026-06-04, mood=1, alcohol=2, start=1`
  対応キー：`mood / cycle / sleep / depas(0|1) / alcohol / body / start(=1) / symptom`
- 保存・取り込み時にシートへも POST（追加 or 同日上書き）。
- 「シートから読み直す」＝シートを正としてローカルを全置換（重複や削除の掃除に使う）。
- 予報ロジック：現状は気圧の前日差のみ。−4hPa 以下=高 / −1.5〜−4=中 / それ以外=低。
  周期・相関・飲酒は表示のみで判定には未反映。

## 既知の教訓

- タイムゾーン：シートが日付を Date 型で保存 → JSON 化で UTC になり前日へずれる。
  アプリ側で全てローカル(JST)日付に変換済み（`localYMD` / `parseYMD` / `pickDate`）。
- 日付列の取り違え：パーサは「日付」列を最優先で読む（A 列「タイムスタンプ」と混同しない）。
- localStorage はオリジン / 端末 / ホーム画面アプリごとに別保管庫。母艦をシートにして解決。
- iPhone はホーム画面追加アプリのキャッシュが強い。Chrome タブで開くと安定。
